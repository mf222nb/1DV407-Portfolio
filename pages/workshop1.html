Så som jag har tänkt med denna modell är att på medlemsklassen så har jag roll med som ett attribut för att skilja på de som är vanliga medlemmar och de som har lite mera ansvar i klubben som sekreterare som kan regristrera händelser till exempel. Alla medlemmar kan bara ha ett inlogg och de kan regristrera hur många båtar som helst. I bokningsklassen har jag gjort så att de attribut som ligger där i är metoder som gör lite olika saker, till exempel ta bort båt, redigera båt, boka händelse och så vidare. Vid sidan av bokning så står det vad de olika metoderna kräver för att kunna fungera så till exempel regristrera båt så behöver man vara medlem för att göra och för att boka en händelse så måste man vara sekreterare. Händelse klassen har jag kopplat till bokning då det finns ett attribut i bokning som heter book event och då tyckte jag att de skulle vara kopplade till varandra men det är bara sekreteraren som kan boka händelser. Kalender klassen har samma attribut som Event klassen för att den ska visa alla medlemmar i klubben vilka händelser som inträffar och den skulle då kunna ärva alla attribut från Event klassen.
